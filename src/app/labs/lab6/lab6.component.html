Formulare HTML

Pentru a permite utilizatorilor să introducă input în paginile web, HTML dispune de mai multe elemente.

Elementul form

Definind un formular, acesta poate conține mai multe elemente HTML pentru input ale căror valori pot fi transmise către un server. Acesta are următoarele atribute specifice:

  • action - determină la ce adresă vor fi trimise datele
  • target - determină în ce fereastră va fi afișat rezultatul (aceleași valori ca în hyperlink)
  • method - determină prin ce metodă vor fi transmise datele

Elementul input

Permite introducerea de către utilizator a mai multor tipuri de date. Modul cum arată și ce date acceptă acest element este determinat de atributul său type:

  • - Permite introducerea de text
  • - Permite introducerea de numere
  • - Permite introducerea de parole. Caracterele introduse în acesta sunt ascunse, având toate formă de disc.
  • - Permite introducerea de e-mailuri. Textul introdus este validat de browser pentru a fi asigurată calitatea sa de e-mail.
  • - Permite introducerea de date.
  • - Permite introducerea de date și timpuri.
  • - Permite încărcarea de fișiere
  • - Afișează un checkbox
  • - Afișează un radio button
  • - Input ascuns ce poate avea orice valoare
  • - Buton ce resetează formularul la valorile inițiale
  • - Buton ce trimite formularul

Elementul textarea

În cazul în care se dorește introducerea unui text mai lung, elementul poate fi de folos. Acesta permite scrierea textului pe mai multe linii, spre deosebire de care afișează textul pe o singură linie.
Lungimea și înălțimea elementului pot fi setate folosind atributele cols și rows (sau folosind CSS).

Elementul select

Pentru a permite utilizatorului să aleagă dintr-un set de opțiuni prestabilite, putem folosi elementul . Opțiunile vor fi definite folosind elemente în care vom specifica prin atributul value valoarea pe care elementul o va lua la selectarea opțiunii respective.

Atașarea evenimentelor

Există două moduri în care putem atașa evenimente la elementele din pagina noastră HTML.

Setând atributele legate de evenimente ale elementor

Toate obiectele din DOM au atribute de tipul on*, unde * reprezintă numele evenimentului pe care vrem să-l atașăm. De exemplu, pentru a atașa un eveniment de click, vom scrie .

Folosind addEventListener()

Spre deosebire de prima metodă, care ne permite adăugarea unui singur event listerer, metoda addEventListener() permite adăugarea de event listeners multiple pe același tip de eveniment. În plus, ne permite să modificăm comportamentul implicit cu care JavaScript tratează evenimentele. Datorită acestor avantaje, folosirea acestei metode este recomandată.
Putem folosi această metodă pe elemente sau pe obiectele document sau window.

Sintaxă:

  • event - String ce reprezintă numele evenimentului
  • listener - Funcție care să fie executată la declanșarea evenimentului
  • useCapture - (opțional) Valoare booleană ce specifică dacă se va folosi event capturing

Pentru a înțelege conceptul de event capturing, este necesar să înțelegem event bubbling, modul implicit prin care funcționează evenimentele în JavaScript. În momentul în care un eveniment se întâmplă pe un anume element, acesta se va propaga mai departe către părinții acestuia, până va ajunge la rădăcina DOM-ului. În schimb, în cazul event capturing, evenimentul se va propaga invers, de la elementul pe care s-a atașat până la ținta efectivă a evenimentului (de la currentTarget la target). Ne putem folosi de acest comportament la evenimentele care nu suportă event bubbling, precum focus.

Observați că pentru elementul ce nu are useCapture, focusarea pe un copil al acestuia nu determină apelarea funcției de logare.

Înlăturarea event listeners

Pentru evenimentele înregistrate cu addEventListener() se poate aplica metoda removeEventListener() pentru a înlătura listener-ul respectiv. Sintaxa acestei metode este exact la fel ca a celei pentru adăugare. Este esențial să păstrăm exact aceiași parametri la ambele, altfel listener-ul nu va fi înlăturat. De exemplu, dacă am folosit element.addEventListener('click', doSomething, true) vom înlătura cu element.removeEventListener('click', doSomething, true).

Teme

Tema 6.1

Creați folosind JavaScript zece elemente în formă de pătrat care să conțină cifre de la 0 la 9. La apăsarea unei taste numerice pe tastatură, schimbați culoarea elementului corespunzător. La ridicarea degetului de pe tastă, culoarea elementului se va întoarce la culoarea inițială.

Tema 6.2

Creați un fișier HTML care să conțină un pătrat. Folosind evenimente ale tastaturii, faceți în așa fel încât pătratul să poată fi mișcat cu săgețile de pe tastatură. Posibilitatea de a mișca pătratul poate fi oprită și repornită apăsând cu mouse-ul pe acesta.

Tema 6.3

La fiecare click dreapta pe ecran va fi afișat într-un element numărul de ordine al click-ului. Elemenul va fi poziționat la coordonatele click-ului și va avea o culoare aleatorie. Acesta va dispărea la următorul click. Meniul obișnuit de context ce se deschide la click dreapta va fi suprimat.

Tema 6.4

Creați un fișier HTML cu un formular HTML. Acesta va conține un formular HTML cu două input-uri de tip număr, un select și un buton de trimitere a formularului. Valorile selectului vor fi operațiile aritmetice obișnuite (adunare, scădere, înmulțire, împărțire, ridicare la putere). La trimiterea forumularului, va fi adăugat în DOM un element ce afișează operația și rezultatul acesteia.

Tema 6.5

Creați un formular și un buton "Adaugă". La apăsarea butonului se va adăuga în formular un element input de tip text de fiecare dată când e apăsat. La scrierea de valori de culoare (de exemplu coduri hex, rgba, etc.) în inputuri, cutia inputului se va colora după codul de culoare respectiv (în cazul în care acesta este valid). Colorarea se va face de îndată ce utilizatorul ridică degetul de pe taste.
